
MetaBlooms Phase 2.1 — Unified Learning Lifecycle (v2.1.1)

This lifecycle defines a strict pipeline. Learning engines do NOT compete; they hand off.

PIPELINE ORDER (Authoritative):
1. In-Turn Learning (repair only)
2. NanoBloom (durability encoding)
3. Experiential Learning (exercise + validation)
4. Telemetry (observation only)
5. MacroBloom (batch-only consolidation)

Only one engine may mutate behavior per pass.
Later engines may observe or defer, never override.

Signal Classification → Eligibility → Arbitration → Execution → Telemetry → Persistence
