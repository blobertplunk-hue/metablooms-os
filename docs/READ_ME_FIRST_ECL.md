# READ ME FIRST — Extraordinary Coding Clarity Layer (ECL) v1

This layer exists so a human or an LLM cannot be confused about what a file does.

How to read any file:
1) Read the ECL header block (what it does / does not do / inputs / outputs / failure modes).
2) Read the module docstring sections in order.
3) Use Examples to confirm call shape.
4) Use Evidence & Observability to know which artifacts prove it worked.

If any part is missing, the file is non-compliant under EXCODE_ECL_V1.
